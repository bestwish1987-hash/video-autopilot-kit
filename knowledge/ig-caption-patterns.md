# IG Reels Caption 模板庫（旅遊+美食帳號實證）

> 來自 video-autopilot-kit 開源知識庫 · MIT 授權

> 用途：寫 IG Reels caption 時自動載入；搭配主 playbook 的 INSTAGRAM REELS 區

---

## 三種 Archetype 對比

| 維度 | A. 景點打卡 | B. 美食探店敘事 | C. 菜單+評分系統 |
|---|---|---|---|
| Caption 長度 | 中（資訊密集） | 長（敘事體驗） | 長（菜單+逐菜評） |
| 第一人稱 | 少 | 多 | 多 |
| 情緒 | 客觀 / 嚮往 | 主觀 / 共鳴 | 權威 / 推薦 |
| Hashtag | 4 個（含英文國際） | 5 個（地理+品類） | 5 個（含系列 hashtag） |
| 結尾錨 | Plus Code 座標 | 營業時間 | 🏠⏰☎️🗾🚇 emoji 圖標分行 |
| 目標 KPI | **Save**（收藏） | **Comment + DM share** | **Save 極致**（高 Save/Like 比） |
| 系列 brand asset | 無 | 影片字卡（系列標題） | 月亮評分系統 🌕🌕🌕🌕🌗 + 系列 hashtag |

## Archetype A：景點打卡

### 數據觀察（3 篇樣本）

| 帖 | Like | Comment | Share | Save | Save/Like |
|---|---|---|---|---|---|
| 水圳絕景 | 萬級 | 百級 | 百級 | 數百 | 約 6% |
| 某半島景點清單 | 數千 | 數十 | 數十 | 數百 | **約 15%** |
| 濱海步道絕景 | 數千 | 數十 | 百級 | 千級 | **約 14%** |

**洞察**：
- 清單型（多景點打包）+ 絕景型 → Save/Like 可達 14% 級
- Hook 觸發詞：「最美的」「不可思議的」「你聽說過」→ 直接撩 save 衝動
- Save 越高 → IG 越擴推（save = 強烈 intent 訊號，2025-2026 演算法重點）

### Caption Anatomy

1. **一句 hook**（caption 主文，貼在 username 旁）— 撩起好奇 / 嚮往
2. 📍 **中文地名**（縣市。具體景點）
3. **English location 一行**（國際 reach + SEO）
4. **背景/規格**（歷史 / 長度 / 工法 — 2-3 行）
5. **特色亮點**（一句拉重點）
6. **限制提醒**（什麼不能做 / 時間）
7. **實用區**（`□` 或 `- ` 條列）：開放時間 / 地址 / Plus Code
8. **Hashtag 一行 4 個**：`#縣市景點 #具體地點 #別名 #英文縣市`

### Template Skeleton

```
{一句 hook，例：不可思議的清澈 / 看看北部最美的海}

📍 {縣市}。{景點}

{English Location Name}

{背景/規格 2-3 行}

{特色亮點 1-2 行}

{限制提醒 1-2 行}

開放時間：{HH:MM-HH:MM，季節變動}
-
□ 地址：{完整地址}
□ 座標：{Plus Code} {里} {縣市}

#{縣市}景點 #{地點} #{別名} #{english_city}
```

## Archetype B：美食探店

### Caption Anatomy

1. **影片開頭字卡**（系列感標題）— 帳號 brand asset，例：「Traveling in {地區} - {主題副標} -」
2. **影片內 text overlay 標重點**（例：「一樓是個人迷你鍋」）— 對應 caption 內容，給靜音觀眾完整資訊
3. **BGM 標註**（音樂選曲，IG 自動顯示）— 強化氛圍
4. **開場定位**（店名 + 一句場景）— 例：「{區域}大名店『{店名}』」
5. **體驗敘事**（第一人稱長段）：
   - 環境氛圍
   - 餐點細節 + 價格
   - 服務互動（具體動作，不只形容詞）
   - **中肯結論**（好+壞都寫 — 真實感是訊號）
6. **結尾 emoji + 真誠評語**（多用 😂）
7. 📍 **店名 + 地址 + 營業時間**
8. **Hashtag 一行 5 個**：`#縣市 #縣市美食 #縣市美食地圖 #縣市晚餐 #區域美食`

### Template Skeleton

```
{店名 / 一句場景定位}
{營業特色 / 客流 hook}（例：一個禮拜只營業三天 / 餐期一堆人在等）
{空間描述}（例：一樓個人鍋二樓共鍋）
{餐點細節 + 價格}（例：牛豬羊三種肉一盤配料 $360）
{服務互動描述 — 越具體越好}
（例：坐好後阿姨來炒肉加湯 / 老闆會問你有沒有吃飽）
{中肯結論 — 好+壞都寫}
（例：吃完中規中矩沒特別驚艷 / 但用餐過程是個蠻特別的體驗）{emoji 😂}

📍 {店名}
•{地址}
•{營業時間}

#{縣市} #{縣市}美食 #{縣市}{餐種} #{縣市}晚餐 #{縣市}餐廳
```

### Reference Quotes（已泛化為示意樣本）

**樣本 1：茶樓/泡茶聊天型**

> 港式中式都有也有熱炒料理

>
> 📍 {店名} • {完整地址} • 10:00-00:00

**樣本 2：石頭火鍋/共鍋型**

> 一個禮拜只營業三天五六日

### 關鍵成功因子

- 🔑 **不假裝萬物皆讚** —「中規中矩沒特別驚艷」「價格偏高吃起來普通」反而推升信任
- 🔑 **互動細節 > 形容詞** —「阿姨炒肉加湯 / 老闆問吃飽沒」比「服務很好」有畫面 100 倍
- 🔑 **系列感字卡** —「Traveling in {地區}」= 帳號 brand asset，每篇連動
- 🔑 **影片內 text overlay** 對應 caption 重點 — 給靜音觀眾完整訊息

## Archetype C：菜單+評分系統化

### 數據觀察（1 篇樣本 — 平價披薩店）

| 帖 | Like | Comment | Share | Save | Save/Like |
|---|---|---|---|---|---|
| 平價披薩店 | 萬級 | 數十 | 數百 | **萬級** | **約 83%** 🔥 |

⚠️ **Save/Like 83% 是極端值** — 菜單+評分+結構化地址 = Save 神器極致版

### Caption Anatomy

1. **權威 hook**：「好吃到我一個月回訪五次！」（行為證據 >>> 形容詞）
2. **發現故事**：「要不是在地朋友帶路...」（in-group 信任）
3. **平價+品質 + emoji 強化**（😻😻）
4. **結論前置**：「不會想吃外面連鎖店了」（給 save 一個強動機）
5. **菜單列表**（◆ 橘菱形 emoji 列點 5 項）
6. **視覺評分**：🌕🌕🌕🌕🌗 (4.5/5) — **月亮系統 = 帳號 brand asset，可重複收藏**
7. **逐菜評論段 + emoji 強化**（😻😻 / 👍👍）— 每道一段
8. **權威排序**：「我吃到的前三名」— 信任拉滿
9. **結構化店家資訊**（emoji 圖標分行 — 視覺易讀 +5x）：
   - 🏠 店名
   - ⏰ 營業時間（公休日）
   - ☎️ 電話
   - 🗾 地址
   - 🚇 最近捷運站
10. **分店補充**：（其他區也有分店）
11. **Hashtag 含系列**：`#{帳號系列 hashtag} in {地區}` + 4 個地理品類
12. **協作 tag**：圈朋友拓散，雙池推播

{結論前置 — 給 save 強理由}
-
◆{菜名1} ◆{菜名2} ◆{菜名3} ◆{菜名4} ◆{菜名5}
整體美味程度：🌕🌕🌕🌕🌗 (4.5/5)
{逐菜評論段，每道一段 + emoji}
{權威排序，例：我吃到的前三名}

🏠：{店名}
⏰：{營業時間}（{公休日}）
☎️：{電話}

## Cross-Pattern 共同點

- 📍 開頭固定地點 emoji（IG 用戶習慣訓練成功）
- 都用「換行分塊」破閱讀牆
- BGM 沉浸式氛圍音（不靠口白）
- Caption 主文（username 旁那行）= 第二個 hook，必定 ≤15 字

## 應用到自己的頻道（依內容類型對位）

- **旅遊 Vlog Reels** → 套 **Archetype A**（如有座標習慣可保留 Plus Code）
- **工具/技巧體驗 Reels** → **Archetype B** 變體：
  - 標題字卡保留（「{主題} 系列」系列感）
  - 體驗敘事 → 改「我用 X 做 Y 遇到什麼」
  - **不要套「正反評價」公式到工具教學** — 工具沒踩坑就不寫缺點，硬寫變假
- **強推薦類教學 / 軟體評測 / 課程介紹 Reels** → **Archetype C**（Save 神器）：
  - 套月亮評分系統 🌕🌕🌕🌕🌗 變 brand asset
  - 用 ◆ 列點功能/重點 5 項
  - emoji 圖標分行（🏠 軟體名 / ⏰ 學習時數 / 💰 價格 / 🔗 連結 / 🎯 適合誰）
  - 系列 hashtag：`#{你的系列} in {領域}` 或類似
- **教學頻道 Shorts repurpose** → 套 A 的清單型（save 神器）：「3 個 {軟體} 你不知道的功能」
